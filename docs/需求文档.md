Template Contract: xian-harness/docs/requirement

## 当前需求

- xian-commit 仅提供工程级安装，每个目标工程分别安装、验证和更新。
- Installer 更新必须保留目标工程 `.xian-commit/config` 与首次原始 hook 备份。
- `verify` 只判断安装资源健康，不把目标工程自定义提交类型误判为安装失败。
- 中英文 README 必须同步说明首次安装、更新路径和外部 hook 管理器边界。

## 需求纪律

- 新需求通过 governed change 定义 scope、non-goals 与 acceptance criteria。
- 不把 Harness 状态、一次性 push 授权或运行日志写入项目 policy。
- 面向开源用户的行为变更同步维护 `README.md` 与 `README_EN.md`。
